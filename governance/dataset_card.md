## 1. 数据集概要

NovelForge-DEMO 用于演示“大女主短篇创作系统”的 AI-ready 数据生产流程，包括作品级规范化、任务样本派生、编辑偏好、盲测评估和反馈回流。它不是可直接商用的小说语料库。

目标任务：

- 创作简报 → 人物卡 / 故事圣经；
- 故事圣经 → 节拍表；
- 节拍 → 场景草稿；
- 首屏钩子、冲突升级、人物一致性、节奏和结局的定向改写；
- 候选稿成对偏好；
- 独立黄金评测与完整故事挑战集。

## 3. 数据来源与许可

允许来源仅包括：公版复核、团队原创、明确授权、正式采购或委托创作。每个作品必须在 `rights_ledger` 中独立记录训练、评测、生成、发布、商业衍生和原文再分发权限。

禁止来源包括：

- 未经许可抓取的付费或平台小说；
- 只有“公开可见”但无训练授权的数据；
- 无法确认作者或权利人的转载文本；
- 含未授权用户隐私、聊天记录或客户数据的内容；
- 处于撤回、争议、过期状态的作品。

## 4. 数据单元与预期制品

| 层级 | 主键 | 说明 |
|---|---|---|
| 权利记录 | `rights_record_id` | 权利基础、用途、期限和证据 |
| 作品 | `work_id` | 一篇完整短篇或独立作品 |
| 文档 | `document_id` | 原始或规范化后的文档实例 |
| 任务样本 | 逻辑键见各制品 Schema | SFT 使用 `sample_id`；偏好使用 `pair_id` 并携带 `sample_id`；评测使用 `eval_id` 并携带 `sample_id` |
| 数据制品 | `dataset_version + shard_id` | 可复现的数据集版本与分片 |
| 训练运行 | `training_run_id` | 使用明确数据版本的训练任务 |
| 模型 | `model_version` | 可反查训练运行和数据版本 |
| 评测 | `eval_version` | 固定任务、裁判规则和报告 |

建议制品：

- `works.jsonl`：作品与文档级元数据；
- `sft.jsonl`：监督微调样本；
- `preferences.jsonl`：成对偏好样本；
- `eval_tasks.jsonl`：独立评测任务；
- `edit_events.jsonl`：编辑修改和耗时；
- `dataset_manifest.json`：数据版本、分片、哈希和统计。

具体字段见 `data_dictionary.md`。

## 5. 处理流程

```text
权利登记
→ 原始文件隔离
→ 文本解析与规范化
→ SHA-256 精确去重
→ MinHash/LSH 近似去重
→ 内容与隐私过滤
→ 先按作者/作品切分
→ 派生 SFT/偏好/评测任务
→ Schema 与泄漏检查
→ 版本固化
→ 训练/评测/反馈关联
```

任何片段化和任务派生都必须发生在作者/作品级切分之后，避免相邻章节、同一作品或同作者风格跨集合泄漏。

## 6. 数据划分

- 推荐按 `author_id` 优先分组；作者不可用时至少按 `work_id` 分组。
- 当前 DEMO 只使用 `train / validation / evaluation`；三者之间不得共享 `work_id`。
- `challenge` 是未来可选的独立评测层，不属于当前机器 Schema。
- 黄金评测集不得用于提示模板调试、SFT、偏好训练或模型选择之外的反复人工优化。
- 切分随机种子、算法版本和输入作品清单必须写入 manifest。

## 7. 质量门禁

发布数据版本前至少满足：

- 权利必填字段完整率：`100%`；
- 已到期、撤回中、争议中作品进入发布集：`0`；
- 主键重复：`0`；
- 精确重复跨 split：`0`；
- 近似重复跨 split：默认 `0`，例外必须审批；
- 作者或作品跨 split 泄漏：`0`；
- Schema 错误：`0`；
- 未处理高风险 PII：生产目标为 `0`；当前合成数据不含真实 PII，但尚未实现生产级自动检测，不能据此声称 PII 门禁已经落地；
- 样本可反查到 `work_id`、`rights_record_id` 和内容哈希：`100%`。

完整契约见 `data_quality_contract.md` 和 `data_quality_contract.yaml`。

## 8. 标注与评测

质量维度包括首屏钩子、女主能动性、因果一致、人物一致、冲突升级、反转兑现、结局闭合、语言质量、多样性和版权记忆风险。

- 模型裁判只用于预筛；
- 最终模型结论使用随机化、去模型标识的成对盲测；
- 允许 `平局` 和 `无法判断`，不得强迫标注员虚假区分；
- 编辑修改耗时以系统事件记录为准，不以回忆估计；
- 报告必须给出样本量、置信区间、无效票和裁决率。

规范见 `annotation_and_blind_eval_protocol.md`。

## 10. 已知限制

- 小说质量具有题材、平台和编辑偏好依赖，单一编辑标签不能代表全部读者。
- 公开热度是受分发、作者影响力和营销影响的弱标签，不能直接等同作品质量。
- 合成数据可能降低多样性并放大教师模型偏差。
- 自动版权记忆风险检测只能提示风险，不能证明不存在侵权。
- 小规模数据上的效果不能外推为“爆款预测”能力。

## 11. 版本与撤回

数据版本采用不可变版本号；任何内容变更都创建新版本。通过：

`rights_record_id → work_id → document_id/content_hash → 各制品逻辑键 → dataset_version/shard_id → training_run_id → model_version`

完成正向追踪和反向影响分析。撤回流程见 `removal_ledger_demo.csv`。
