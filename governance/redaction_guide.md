# 脱敏与对外展示指南

> 状态：`DEMO / 操作规范`  
> 原则：证明公开 Demo 的工程产物可以运行和核验，同时不暴露任何真实作者、用户、客户、合同、账号和原文。

## 1. 数据分级

| 级别 | 示例 | 对外处理 |
|---|---|---|
| 公开 | 自有架构图、空白 Schema、公开代码 | 可展示 |
| 内部 | 聚合质量指标、无标识错误案例 | 审批后展示 |
| 机密 | 小说全文、提示词资产、平台后台明细 | 不进入公开仓库 |
| 严格机密 | 合同、身份证、银行卡、客户名单、访问令牌 | 只存受控证据库 |

## 2. 必须删除或替换

- 姓名、笔名、手机号、邮箱、身份证件和精确地址；
- 平台账号、店铺、作品号、订单号、合同相对方名称；
- 银行账号、支付流水号、税号、发票号码；
- URL 中的访问令牌、签名参数和私有对象路径；
- 未发布正文、长段落和可搜索的独特句子；
- 截图中的浏览器资料、系统用户名、通知、聊天侧栏和二维码。

替换为稳定的不可逆化名，如 `AUTHOR-7F31`、`WORK-A204`。不要使用姓名首字母等可猜测规则。

## 3. 哈希与证据链

- 原始文件保存在受控证据库；
- 对原始证据计算 SHA-256；
- 对外仓库只保存 `evidence_ref`、哈希、采集时间和复核人化名；
- 截图脱敏后再次计算哈希，并明确“原始哈希”和“脱敏副本哈希”；
- 哈希只能证明文件一致性，不能单独证明内容真实或权利有效。

## 4. 金额与业务指标

可以展示：

- 经审批的月度聚合区间；
- 脱敏后的支付凭证局部；
- 可由后台导出或财务记录复核的计算口径。

不得：

- 用合同额代替实际到账；
- 用总流水代替净收入；
- 把退款前金额包装成收入；
- 把 DEMO 金额换成“真实案例”；
- 为了简历效果补造用户数、发布量或模型胜率。

## 5. 小说文本脱敏

- 优先展示自写的合成短样例，而非真实训练文本；
- 需要展示编辑 diff 时，使用获得展示许可的短片段并控制长度；
- 替换专有名词后检查情节组合是否仍可被搜索定位；
- 版权风险案例仅展示哈希、相似度和处理决定，不展示大段命中文本。

## 6. 截图检查清单

1. 裁剪到证明所需的最小区域；
2. 实心覆盖敏感内容，不能只做可逆模糊；
3. 检查图片 EXIF 和文件名；
4. 检查二维码、头像、通知栏和浏览器标签；
5. 在角落标注 `REDACTED` 或 `DEMO`；
6. 保留原始证据和脱敏副本的哈希；
7. 由第二人复核后才对外使用。

## 7. 对外证明状态

每条证明必须标记：

- `VERIFIED`：存在可复核的真实证据；
- `REDACTED_VERIFIED`：真实证据已核验，对外版本完成脱敏；
- `UNVERIFIED`：只有陈述，尚无证据；
- `DEMO`：完全虚构，只用于展示格式；
- `TEMPLATE`：空白模板。

这些状态只适用于对外声明和证据文件，不是数据质量、授权有效性、发布审核或撤回处理状态。权利台账、数据质量门禁和发布流程必须使用各自独立的状态字段，不能用 `VERIFIED` 代替“授权可训练”或“质量通过”。
