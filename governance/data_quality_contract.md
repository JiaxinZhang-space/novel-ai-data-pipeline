# 数据质量契约

> 状态：`TARGET_CONTRACT / PARTIALLY_IMPLEMENTED`  
> 机器可读规则见 `data_quality_contract.yaml`。

当前管道只执行并报告该目标规范的一个子集：合成权利案例硬门、生成制品 Schema 测试、精确正文/作者/作品泄漏检查，以及训练与评测隔离。通用主键唯一性、跨集合近重复、生产级 PII、完整可追踪率量化、软门禁和审批流仍是目标设计，不能把 YAML 中列出的规则全部表述为已经执行。

## 1. 契约目标

本契约用于决定一个数据版本能否进入训练、评测或发布流程。检查结果分为：

- `BLOCK`：立即阻断，不得人工口头放行；
- `REVIEW`：需要数据与编辑负责人书面审批；
- `WARN`：允许发布，但必须进入报告；
- `PASS`：满足要求。

## 2. 硬门禁

| 编号 | 检查 | 阈值 | 失败级别 |
|---|---|---:|---|
| R01 | `rights_record_id` 可解析且授权有效 | 100% | BLOCK |
| R02 | 训练、评测或发布用途与当前制品用途匹配 | 100% | BLOCK |
| R03 | 到期、撤回中、争议中作品数 | 0 | BLOCK |
| S01 | 必填字段完整率 | 100% | BLOCK |
| S02 | Schema 类型和枚举错误 | 0 | BLOCK |
| U01 | 各制品逻辑主键重复数：SFT=`sample_id`，偏好=`pair_id`，评测=`eval_id` | 0 | BLOCK |
| D01 | 规范化正文精确重复跨 split | 0 | BLOCK |
| D02 | 高相似文本跨 split | 0 | BLOCK |
| L01 | `work_id` 跨 split | 0 | BLOCK |
| L02 | 已知 `author_id` 跨 split | 0 | BLOCK |
| L03 | 评测污染命中数 | 0 | BLOCK |
| P01 | 未处置高风险 PII 命中数 | 0 | BLOCK |
| T01 | 样本到权利记录和内容哈希可追踪率 | 100% | BLOCK |

## 3. 软门禁

阈值必须通过真实基线确定，下列仅为起始建议：

| 检查 | 建议阈值 | 处理 |
|---|---:|---|
| 单一作者占训练字符比例 | ≤ 10% | REVIEW |
| 单一来源占训练字符比例 | ≤ 30% | REVIEW |
| 合成样本占训练样本比例 | 分实验桶记录，不设伪科学固定值 | REVIEW |
| 文本过短/过长异常比例 | ≤ 2% | WARN |
| 编辑拒绝样本混入 SFT | 0 | BLOCK |
| 偏好对 `abstain` 比例 | 报告实际值 | WARN |
| 标签原因码缺失 | 0 | BLOCK |

## 4. 去重与泄漏定义

- 精确重复：规范化文本 SHA-256 相同。
- 近似重复：同一分词与 MinHash 配置下，LSH 候选经二次相似度计算超过版本阈值。
- 作品泄漏：相同 `work_id` 出现在多个 split。
- 作者泄漏：相同已知 `author_id` 出现在多个 split。
- 评测污染：训练候选与评测提示、`reference_answer` 或其他隔离评测文本达到精确或近似匹配阈值。

逻辑标识统一规则：SFT 以 `sample_id` 为主键；偏好训练对以 `pair_id` 为主键并携带 `sample_id`；评测以 `eval_id` 为主键并携带 `sample_id`。不得声称所有制品主键都叫 `sample_id`。

阈值、分词器、n-gram、签名数和随机种子必须进入 manifest，不能只在报告中描述。

## 5. 发布报告最小字段

生产版本的每次质量检查应输出：

- `run_id`、代码提交或规范化源码快照哈希、输入/输出数据版本；
- 检查规则版本；
- 输入量、输出量、保留率；
- 每类过滤原因计数；
- 精确与近似重复计数；
- 各 split 的作品、作者、样本和字符数；
- 泄漏命中及处理结果；
- 失败规则、审批人和证据引用；
- 最终状态 `PASS/BLOCKED`。

## 6. 例外处理

任何 `REVIEW` 例外必须记录：

1. 规则编号；
2. 影响范围；
3. 业务理由；
4. 风险缓解措施；
5. 到期时间；
6. 数据、编辑和权利复核人的审批。

硬门禁不得通过修改报告文字绕过；需要修复数据并生成新版本。
