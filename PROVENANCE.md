# 合成数据来源说明

## 公开仓库来源边界

本仓库的公开数据全部由 `src/novel_evidence/generator.py` 中的确定性模板、角色名、职业、冲突和结局字段组合生成。生成器固定随机种子和缺陷注入规则，以便测试权利拒绝、精确重复、近似重复和集合泄漏检查。

- 未导入任何朋友、公司、客户或平台项目的代码、正文、日志、提示词、配置、合同或经营记录；
- 未抓取或改写第三方小说；
- 作者、作品、合同、审核人、编辑会话、实验和版本标识均为虚构；
- `artifacts/` 可由生成器完整重建，不是生产系统导出；
- `experiments/demo/` 是格式和对账样例，不是实际模型运行。

## 生成路径

```text
src/novel_evidence/generator.py
  -> 27 条合成原始记录（含故意注入的无效权利与重复缺陷）
src/novel_evidence/pipeline.py
  -> 权利硬门、规范化、去重、切分和数据集派生
artifacts/
  -> 合成数据、指标、离线报告、校验和与版本关系
```

每个生成记录和核心运行产物都必须携带 `demo=true`；CI 中的 `scripts/check_public_repo.py` 会阻断缺少 DEMO 标记的 JSON / JSONL 产物及常见敏感文件。

## 工具使用披露

该作品集使用 AI 编码助手加速文档起草、实现和审计。仓库维护者负责确定业务边界、选择工程方案、运行测试、复核输出、解释设计取舍，并对最终公开内容承担维护责任。AI 辅助不等于真实业务经历或第三方背书。
