# 项目证据索引

> 状态：`TEMPLATE`  
> 作用：把简历中的每个数字关联到可复核证据，避免“报告里有数字、但找不到来源”。

## 1. 状态定义

- `VERIFIED`：真实证据已由指定复核人核验；
- `REDACTED_VERIFIED`：真实证据已核验，展示副本已脱敏；
- `UNVERIFIED`：尚无充分证据，不得写成已完成；
- `DEMO`：虚构演示；
- `TEMPLATE`：空白结构。

## 3. 推荐声明编号

| claim_id | 指标口径 | 最低证据 |
|---|---|---|
| `CLAIM-DATASET-SIZE` | 某数据版本验收样本数 | manifest + 质量报告 |
| `CLAIM-BLIND-WINRATE` | 固定 eval 版本盲测胜率 | 原始标签 + 随机化记录 + 报告 |
| `CLAIM-EDIT-TIME` | 同条件下活跃编辑耗时变化 | 工作台事件导出 + 统计脚本 |
| `CLAIM-PUBLISHED-COUNT` | 实际发布且未撤回作品数 | 平台记录 + 内容映射 |
| `CLAIM-ACTIVE-USERS` | 口径明确的活跃使用人数 | 去重规则 + 事件导出 |
| `CLAIM-REVENUE` | 指定期间实际确认收入 | 财务口径 + 到账/结算凭证 |

## 4. 完整性要求

每条对外声明必须回答：

1. 数字对应哪个时间范围？
2. 分母、去重规则和排除项是什么？
3. 来自哪个数据、模型和评测版本？
4. 原始证据在哪里，谁核验过？
5. 对外副本删除了哪些信息？
6. 是否存在退款、撤回、无效任务或其他不利事实？
